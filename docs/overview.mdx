## Samples {#samples}

The home page "Samples" contains all samples that are uploaded into patients or runs that are not in the archive.
If a sample has both a patient and a run defined, it will appear on the "Samples" page unless at least one of
these entities is archived.

<details>
<summary>Click to learn more about the way information is presented on the "Samples" page.</summary>

#### On the "Samples" page, information is presented as a table with the following columns: {#sample-columns}
- «SAMPLES» - contains information about samples uploaded into the system that are added to patients or runs
that are not in the archive:
1. Sample processing status:<br></br><img src={require('/img/eng/in_process_sample.png').default} width="18"/> - tumor
sample in progress, queued, or waiting for the prerequisite stages of the control sample to be
completed;<br></br><img src={require('/img/eng/tumor_success.png').default} width="18"/> - successfully processed
tumor sample;<br></br><img src={require('/img/eng/tumor_problems.png').default} width="18"/> - tumor
sample with problems;<br></br><img src={require('/img/eng/tumor_failed.png').default} width="18"/> - failed
tumor sample;<br></br><img src={require('/img/eng/normal_in_process.png').default} width="18"/> - normal
sample in progress or queued;<br></br><img src={require('/img/eng/normal_success.png').default} width="18"/> - successfully
processed normal sample;<br></br><img src={require('/img/eng/normal_problems.png').default} width="18"/> -
normal sample with problems;<br></br><img src={require('/img/eng/normal_failed.png').default} width="18"/> -
failed normal sample.
2. Name of the sample file without extension and postfixes (if files are named according
to <a href="https://support.illumina.com/help/BaseSpace_Sequence_Hub_OLH_009008_2/Source/Informatics/BS/NamingConvention_FASTQ-files-swBS.htm">the Illumina convention</a>).
In the case of paired-end sequencing, only the name of the primary file is given.
You can open <a href="/results/results">the sample page</a> by clicking on the file name.
3. File format: <i>FASTQ</i> or <i>VCF</i>. For samples uploaded in BAM format, or genotyping data uploaded in
TXT, TSV format, the format into which the original data was converted for further analysis will be shown,
i.e. FASTQ and BAM, respectively.
4. Sequencing type: <i>Panel</i> (targeted sequencing), <i>WES</i> (whole-exome
sequencing), <i>WGS</i> (whole-genome sequencing), <i>Low-pass WGS</i> (whole-genome sequencing with low
coverage), <i>Unknown</i> (the sequencing type may not be detected for some samples in VCF format).
Detected automatically by Genomenal, may be based on Sequencing Type and Capture Kit selected while
composing sample set. Displayed only after determintation, i.e. after successful completion of the "Determine Kit"
stage for FASTQ files and the "Identify" or "SNVs/Indels annotation" stages for VCF files.
5. Sample size. In the case of paired-end sequencing, the total size of the sample files will be displayed.
- «PATIENT / RUN» - contains patient ID (indicated by <img src={require('/img/eng/patient_icon.png').default} width="20"/>)
and/or run name (indicated by <img src={require('/img/eng/run_icon.png').default} width="20"/>)
to which the sample belongs. By clicking on the patient ID or the run name, you can
open <a href="/add-patient#patient-page">the patient page</a> or <a href="/runs#run-page">the run page</a>,
respectively.
- «RESULTS» - contains sample interpretation summary: the information on the number of variants of a particular
pathogenicity class discovered in the sample
and <a href="/results/main/snvs-indels/variant-viewer#include-to-report">included to report</a>
(Pathogenic: <img src={require('/img/results/pathogenic.png').default} width="48"/>,
Likely pathogenic: <img src={require('/img/results/likely_pathogenic.png').default} width="48"/>,
Uncertain significance: <img src={require('/img/results/uncertain.png').default} width="48"/>,
Likely benign: <img src={require('/img/results/likely_benign.png').default} width="48"/>,
Benign: <img src={require('/img/results/benign.png').default} width="48"/>).
Results are displayed if sample interpretation has been completed (as described below) or if a variant
with a particular pathogenicity has been included to report.
If the interpretation has not been completed, but variants with a particular pathogenicity class have been
included to report, then the result will be "<i>Interpretation not completed</i>". If the
sample interpretation has been completed, but no variants with a particular pathogenicity class have been
included to report, then the result will be "<i>No pathogenic variants found</i>".
- «CREATED» - contains the date of the beginning of uploading sample data into the system in M/d/yyyy format;
- «MODIFIED» - contains the date of the last changes in the sample data in M/d/yyyy format.

By default, the table is sorted from samples uploaded last to samples uploaded first (i.e., in descending order
of the "Created" column values). The table can be sorted by ascending or descending values of all its columns.
To sort a table by column values, click on its name (click once to sort ascending, or twice to sort descending).
After this, an arrow <img src={require('/img/vv/sort_asc.png').default} width="17"/> will appear next to the column
name (when sorting in ascending order) or <img src={require('/img/vv/sort_desc.png').default} width="17"/> (when
sorting in descending order). To reset the sorting, refresh your browser page. The values of the "Samples", "Patient",
and "Run" columns are sorted alphabetically, and the values of "Created" and "Modified" columns are sorted from
earliest to latest date and vice versa. The sort order for the "Results" column is as follows:
1. Samples that have pathogenic or likely pathogenic variants or variations in the results;
2. Samples that do not have pathogenic or likely pathogenic variants or variations in the results, but the variants
or variations with a different pathogenicity class were added;
3. Samples with the result "<i>No pathogenic variants found</i>";
4. Samples with the result "<i>Interpretation not completed</i>";
5. Samples for which the interpretation has not been completed and which do not have any variants with a
determined pathogenicity that was not added to report or which do not have any variations with a
determined pathogenicity.

When you hover over a sample row, the following active elements appear:
- <img src={require('/img/eng/sample_vv.png').default} width="17"/> - button to
open <a href="/results/main/snvs-indels/variant-viewer">SNV Viewer</a> of the sample. For a tumor sample,
this is SNV Viewer with somatic SNVs/Indels, in which you can also see SNVs/Indels from SNV Viewer with
germline mutations if you change the filter value by origin from Somatic to Germline or clear filtering by it.
For a non-tumor sample, this is SNV Viewer with only germline SNVs/Indels discovered in that sample.
The button is displayed if the "SNVs/Indels annotation" stage has successfully completed for the sample.
- <img src={require('/img/eng/sample_cnvv_segments.png').default} width="22"/> - button to open the sample
CNV Viewer with the segments discovered in that sample. The button is displayed if the "Copy number variations
discovery" stage has successfully completed for the sample.
- <img src={require('/img/eng/sample_cnvv_bins.png').default} width="22"/> - button to open the sample
CNV Viewer with the bins discovered in that sample. The button is displayed if the "Copy number variations
discovery" stage has successfully completed for the sample.
- <img src={require('/img/eng/sample_complete_interpretation.png').default} width="22"/> or <img src={require('/img/eng/sample_restore_interpretation.png').default} width="22"/> - buttons
to complete and resume sample interpretation, respectively.
After clicking the complete button, you will see a confirmation window:

<p align="center">
<img src={require('/img/vv/interpretation_window.png').default} height="600"/>
</p>

After you click on <img src={require('/img/custom-annot/ok_btn.png').default} width="130"/>, interpretation
summary from this window will be displayed in the «RESULTS» column.

- <img src={require('/img/eng/delete_btn.png').default} width="20"/> - button to delete a sample from the
system. After clicking on it, you will see a confirmation window:

<p align="center">
<img src={require('/img/eng/delete_sample.png').default} width="450"/>
</p>

After confirming the action by clicking on <img src={require('/img/eng/delete.png').default} width="150"/>,
the sample will be removed both from the "Samples" page and from the page of the patient and/or run to which it
was added.

At the top of the page there are:
- Search window allowing to search for samples by file name and by the patient ID or run name to which they belong:
<p align="center"><img src={require('/img/eng/search_sample.png').default} width="500"/></p>
- Button to <a href="/add-patient">add a patient</a> to
the system <img src={require('/img/eng/add_patient_btn_from_samples.png').default} width="180"/>.
- Button to <a href="/runs">add a run</a> to
the system <img src={require('/img/eng/add_run_btn.png').default} width="170"/>.

At the bottom of the page there are:
- Sample count information: the order of samples displayed on the page (a maximum of 10 samples can be shown on a
page, so the order would be 1-10, 11-20, etc.) and the total number of samples.
- Pagination component: <img src={require('/img/eng/pagination.png').default} width="400"/>, which allows to
navigate between sample pages. The number of the opened page is highlighted in a blue circle. If you click on
this number, the page will reload. To move to a page with a specific number, either click on that number, or
click on <img src={require('/img/eng/three_points.png').default} width="30"/>, enter the required number and press
the enter key or click outside the value field. To move to the previous page,
click on <img src={require('/img/eng/back.png').default} width="30"/>, and to open the next page,
click on <img src={require('/img/eng/next.png').default} width="30"/>.

</details>

- "***All Patients***" - contains all patients added to the system and their data.<br></br>

- “***Problems***” - contains those patients with data processing problems or with peculiar data.

data for later deletion here. Archived patients are not displayed on the "All Patients" page.

## Cohorts {#cohorts}

button <img src={require('/img/eng/add_cohort.png').default} width="20"/> for adding a cohort.

You can read more about cohorts [here](/cohorts).
If there are no cohorts added, then the section contains a window for grouping patients into cohorts:

<p align="center">
<img src={require('/img/eng/cohorts_empty.png').default} width="170"/>
</p>

A cohort page contains only data of patients included in this cohort:

<p align="center">
<img src={require('/img/eng/cohorts_added.png').default} width="170"/>

element <img src={require('/img/eng/cohort_options.png').default} width="16"/>. If you click on it, you will see
the options "***Edit***" (opens the cohort editing window) and "***Delete***" (deletes the cohort).
Please note that deleting a cohort does not delete the data of patients included in it.
